# VideoInfo Guide

## Overview

The VideoInfo project provides the ability for a user to save the current display layout set in Windows 10/11 to a file, and the load those settings at a later date. The user saves each display layout they want to use to a different JSON display settings file, and can then later choose which display settings file they want to load, at which point the display layout will be restored to how it was when the settings file was saved.

The VideoInfo programme is designed to interact with four different video display APIs that span the most common video APIs in use today:
- The Windows Display API (Both the Windows CCD and GDI APIs)
- The NVIDIA NVAPI (NVIDIA's Video API)
- The AMD ADLX API and the AMD ADL2 API (AMD Display Library eXtended and AMD Display Library 2)
- The Intel IGCL API (Intel Graphics Control Library)

The WindowsLibrary code uses the Windwos Display API is used to provide support for all video cards, as every Windows PC has access to the Windows Display API. 
The NVIDIALibrary uses the NVIDIA NVAPI directly to provide support for additional NVIDIA video card features like NVIDIA Surround multi-display and other special functionality.
The AMDLibrary uses the ADLXWrapper DLL to provide support for the AMD ADLX API, and it uses an earlier ADL2 API to provide access to earlier versions of AMD Eyefinity.
The IntelLibrary uses the IGCLWrapper DLL to provide support for the Intel IGCL API for Intel video cards.

## Project Tech Stack

### Programming Language

This project is written in C# (.NET 10) and uses various libraries like ADLXWrapper and IGCLWrapper with ClangSharp-generated P/Invoke bindings to access various GPU vendor APIs.

#### C#

The project consists of:
- **Program.cs** - Main application code
- **Windows/*.cs** - Windows Display Library abstraction layer, using Windows CCD and GDI Display APIs
- **NVIDIA/*.cs** - NVIDIA Library abstraction layer, using NVIDIA NVAPI
- **AMD/*.cs** - AMD Library abstraction layer, using ADLXWrapper, which abstracts the AMD ADLX API, and the ADL.cs file which accesses the ADL2 APIs
- **Intel/*.cs** - Intel Library abstraction layer, using IGCLWrapper which abstracts the Intel IGCL API


### Build System

This project uses .NET 10.0 SDK and MSBuild for compilation. The project should be able to build within Visual Studio 2022, Visual Studio Code, and via command-line using `dotnet` CLI.

## Architecture

### Overall structure

The project uses modular structure with separate folders for each GPU vendor library, and a common interface (program.cs) for display management. This design was chosen as the main reason for this software is to test the video functionality for DisplayMagician (an opensource video display management tool) across multiple GPU vendors, and so having separate modules for each vendor allows for easier maintenance and updates as.



1. **ClangSharp** reads `igcl_api.h` and generates `IGCL.cs`
2. **IGCL.cs** contains raw P/Invoke declarations for ControlLib.dll
3. **IGCLExtensions.cs** adds extension methods for usability
4. **IGCLApi.cs** provides high-level managed API with error handling
5. Applications use `IGCLApi` class for safe, managed access


### Direct P/Invoke Approach

The project uses a direct P/Invoke approach with no intermediate C++ wrapper:

1. **ClangSharp** reads `igcl_api.h` and generates `IGCL.cs`
2. **IGCL.cs** contains raw P/Invoke declarations for ControlLib.dll
3. **IGCLExtensions.cs** adds extension methods for usability
4. **IGCLApi.cs** provides high-level managed API with error handling
5. Applications use `IGCLApi` class for safe, managed access

### File Responsibilities

1. **IGCL.cs** (Auto-generated - **DO NOT EDIT MANUALLY**)
   - Raw P/Invoke function declarations
   - Struct definitions matching C layout
   - Enums and constants
   - Function pointer delegates
   - **Regenerated automatically on build when headers change**
   - Any manual changes will be lost on next build

2. **IGCLExtensions.cs** (Manual)
   - Extension methods for generated types
   - String marshaling helpers
   - Safe handle wrappers for pointers
   - Convenience methods for common patterns

3. **IGCLApi.cs** (Manual)
   - High-level API facade
   - Exception handling (IGCLException)
   - Resource management (IDisposable pattern)
   - Simplified method signatures
   - Memory safety guarantees

4. **IGCLHelpers.cs** (Manual - if needed)
   - Utility methods
   - Type conversions
   - Common patterns and helpers

## Project and Code Guidelines

### Critical Rules

- **CRITICAL: ALWAYS ASK THE USER FOR PERMISSION IF THE ORIGINAL STRATEGY THE USER APPROVED DOESN'T WORK AND YOU WANT TO CHANGE THE STRATEGY YOU USE TO ACHIEVE THE OUTCOME. THE USER MUST APPROVE THAT CHANGE.**

- **CRITICAL: Never remove any functionality without asking for permission to do so!**
  - This means NEVER remove any functions from any file without user approval

- **CRITICAL: All Video Librarys MUST reamin part of the DisplayMagicianShared namespace!**
  - ALWAYS ensure that ALL files in the root folder are in the VideoInfo namespace
  - ALWAYS ensure that ALL files in the Windows/ folder are in the DisplayMagicianShared.Windows namespace
  - ALWAYS ensure that ALL files in the NVIDIA/ folder are in the DisplayMagicianShared.NVIDIA namespace
  - ALWAYS ensure that ALL files in the AMD/ folder are in the DisplayMagicianShared.AMD namespace
  - ALWAYS ensure that ALL files in the Intel/ folder are in the DisplayMagicianShared.Intel namespace  
  - This is because the DisplayMagician application depends on these namespaces to find the classes it needs to use, and the main purpose oif this project is to test the Video Libraries for DisplayMagician

### General Guidelines

- Never truncate large files. If you find that you have truncated a file and have lost some functionality then restore the file back to where it has the full functionality back and then try again.
- Always use type hints in any language which supports them (C# nullable reference types enabled)
- All public types should be serializable to JSON using System.Text.Json or Newtonsoft.Json
- Unit tests are required, and are required to pass before PR passes
  - Unit tests should focus on core functionality
  - Unit tests should be made in xUnit.net, and should be tested in .NET 10.0
  - Tests are in VideoInfo.Tests project
- Always follow good security practices
  - Use safe pointer handling (SafeHandle where appropriate)
  - Validate all inputs before P/Invoke calls
  - Dispose resources properly (IDisposable pattern)
- Always update documentation if you change functionality
- When using the terminal to execute commands, always remember you are running within a PowerShell Terminal on a Windows 11 PC
- Do not use older Command Terminal commands - use PowerShell or `dotnet` CLI commands instead

## Prerequisites

- **Windows 10/11**
- **.NET 10.0 SDK** - Download from https://dotnet.microsoft.com/download/dotnet/10.0
- **Visual Studio 2022** (or higher) - Community edition or better
  - Workload: ".NET desktop development"
  - Optional: "Desktop development with C++" (for IGCL SDK samples)
- **Intel graphics drivers** with IGCL support (discrete Arc GPUs or recent integrated GPUs)
- **Internet connection** (for initial IGCL SDK download)

**Note:** ClangSharp NuGet packages are automatically restored during build. No manual SWIG installation needed.

## Building the Wrapper

### Prerequisites Check

```powershell
# Verify .NET 10.0 SDK is installed
dotnet --version  # Should show 10.0.x or higher

# Verify NuGet restore works
dotnet restore
```

### Option 1: Using Command Line (Recommended)

```powershell
# 2. Build the VideoInfo project
dotnet build VideoInfo/VideoInfo.csproj

# 3. Run tests
dotnet test VideoInfo.Tests/VideoInfo.Tests.csproj

# 4. Build everything (solution)
dotnet build VideoInfo.sln
```

### Option 2: Using Visual Studio 2022

1. Open `VideoInfo.sln` in Visual Studio
2. Restore NuGet packages (right-click solution → Restore NuGet Packages)
3. Build the solution (Ctrl+Shift+B)
4. Run tests (Test Explorer → Run All)

### Option 3: Using Visual Studio Code

1. Open folder in VS Code
2. Install C# Dev Kit extension
3. Press Ctrl+Shift+B to build
4. Use Test Explorer to run tests


## Testing

### Running Tests

```powershell
# Run all tests
dotnet test

# Run specific test class
dotnet test --filter FullyQualifiedName~CoreApiTests

# Run with detailed output
dotnet test --logger "console;verbosity=detailed"

# Run tests and collect coverage
dotnet test /p:CollectCoverage=true
```

### Test Organization

- Tests are organized by VideoLibrary category.
- Tests use xUnit.net framework.
- Tests use xUnit.skippable for conditional skipping.
- Tests validate core functionality and error handling.
- Tests skip IntelLibrary gracefully if Intel GPU not present.
- Tests skip AMDLibrary gracefully if AMD GPU not present.
- Tests skip NVIDIALibrary gracefully if NVIDIA GPU not present.
- Tests use real hardware access where possible. We have multiple test machines with different GPUs for coverage.

### Writing New Tests

Follow this pattern:

```csharp
[Fact]
public void ApiMethod_Condition_ExpectedResult()
{
    // Arrange
    using var api = IGCLApi.Initialize();
    
    // Act
    var result = api.SomeMethod();
    
    // Assert
    Assert.NotNull(result);
}
```
